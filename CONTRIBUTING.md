# Contributing to Markdown Auditor

First off, thank you for considering contributing.

## How Can I Contribute?

### Reporting Bugs
* Check the existing logs to ensure the bug hasn't been reported.
* Open an issue and describe the bug, including your OS and Python version.

### Suggesting Enhancements
New ideas! Current roadmap goals include:
* **Auto-Fix Mode:** Automatically adding "FIXME" to empty brackets.
* **Link Validator:** Checking if external URLs return a 404.
* **Duplicate Detection:** Finding unused image assets.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the code adheres to **PEP 8** standards.
4. Make sure the `tqdm` progress bar and `loguru` logging still function correctly.

## Style Guide
* Use **PEP 8** for Python code.
* Keep the "nerdy" terminal aesthetic (emojis in logs are encouraged!).
* Update the `README.md` if you add new features.

Happy coding! 😁🐍
